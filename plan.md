# 테트리스 웹 애플리케이션 구현 계획

## Context
사용자가 브라우저에서 바로 실행 가능한 간단한 테트리스 게임을 요청했습니다. 현재 빈 디렉토리이며, MVP 버전으로 최소한의 기능만 구현하면 됩니다:
- 점수 표기
- 블록 이동 (좌우, 회전, 낙하)
- 깔끔한 웹 디자인

## 구현 방식

HTML, CSS, JavaScript를 별도 파일로 분리하여 구현합니다.

### 파일 구조
```
index.html   - HTML 구조
style.css    - CSS 스타일링
game.js      - JavaScript 게임 로직
```

### 주요 컴포넌트

**1. HTML 구조 (index.html)**
- 게임 캔버스 (Canvas API 사용)
- 점수 표시 영역
- 다음 블록 미리보기
- 게임 조작 안내
- 재시작 버튼

**2. CSS 스타일링 (style.css)**
- 중앙 정렬된 게임 레이아웃
- 현대적이고 깔끔한 디자인
- 그라데이션 배경
- 반응형은 고려하지 않음 (MVP)

**3. JavaScript 게임 로직 (game.js)**

**게임 보드:**
- 10x20 그리드 (표준 테트리스 크기)
- 2D 배열로 보드 상태 관리

**테트로미노 (블록):**
- 7가지 기본 도형 (I, O, T, S, Z, J, L)
- 각 블록별 색상 지정
- 회전 상태 배열

**게임 메커니즘:**
- 블록 생성 및 랜덤 선택
- 좌우 이동 (방향키 ←, →)
- 회전 (방향키 ↑)
- 빠른 낙하 (방향키 ↓)
- 즉시 낙하 (Space)
- 자동 낙하 (타이머)
- 충돌 감지
- 라인 완성 감지 및 제거
- 점수 계산 (1줄당 100점)

**게임 상태:**
- 게임 시작/종료
- 게임 오버 감지 (블록이 맨 위에 도달)
- 재시작 기능

### 구현 상세

**Canvas 렌더링:**
```javascript
- drawBoard(): 보드 그리기
- drawPiece(): 현재 블록 그리기
- drawSquare(): 개별 블록 그리기
- clearLines(): 완성된 라인 제거
- drawNextPiece(): 다음 블록 미리보기
```

**입력 처리:**
```javascript
- 키보드 이벤트 리스너
- ArrowLeft: 좌로 이동
- ArrowRight: 우로 이동
- ArrowUp: 회전
- ArrowDown: 빠른 낙하
- Space: 즉시 낙하
```

**게임 루프:**
```javascript
- setInterval 사용
- 500ms 간격으로 블록 자동 낙하
- 게임 상태 업데이트 및 렌더링
```

## 검증 방법

1. **파일 생성 후 브라우저에서 열기:**
   ```bash
   open index.html  # macOS
   xdg-open index.html  # Linux
   ```

2. **기능 테스트:**
   - [ ] 게임이 자동으로 시작되는지 확인
   - [ ] 블록이 위에서 아래로 떨어지는지 확인
   - [ ] 방향키로 블록 이동/회전이 되는지 확인
   - [ ] 라인이 완성되면 제거되고 점수가 올라가는지 확인
   - [ ] 블록이 맨 위에 쌓이면 게임 오버되는지 확인
   - [ ] 재시작 버튼이 작동하는지 확인
   - [ ] 다음 블록 미리보기가 표시되는지 확인

3. **브라우저 콘솔 확인:**
   - JavaScript 오류가 없는지 확인

## 구현하지 않을 기능 (MVP 범위 밖)
- 레벨 시스템
- 홀드 기능
- 고스트 피스 (착지 위치 미리보기)
- 사운드 효과
- 로컬 스토리지 최고 점수
- 모바일 터치 컨트롤
- 복잡한 애니메이션 효과
